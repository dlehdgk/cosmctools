# Package for Cosmological model comparison (and others) from Cobaya MCMC chains

## Installation

Clone this repository and install using `pip` or `uv` (recommended)

for `pip`:

```bash
git clone https://github.com/dlehdgk/cosmctools.git
pip install -e cosmctools
```

for `uv`, recommended that you create a new environment for this package:

```bash
uv init
git clone https://github.com/dlehdgk/cosmctools.git
uv add --editable cosmctools
```

It is recommended that you install this package in editable mode as you may wish to modify the code for your specific choice of models and datasets.

If you wish to add this package to a pre-existing `uv` environment with a full installation of `Cobaya` you may experience some incompatibilities with `harmonic`. It may be resolved by adding the following lines to your `pyproject.toml` file:

```toml
[tool.uv]
override-dependencies = [
    "wheel>=0.47.0",
    "setuptools>=68.0.0",
]
```

## Usage

This is package is essentially an umbrella wrapper for multiple cosmology codes, please ensure that the original authors of the codes are cited if they have been used. This code takes MCMC chains generated by (Cobaya)[https://cobaya.readthedocs.io/en/latest/index.html] developed by (J. Torrado and A. Lewis 2021)[http://arxiv.org/abs/2005.05290] and is able to:

- Return constraints on requested parameters using (getdist)[https://getdist.readthedocs.io/en/latest/] by (A. Lewis 2019)[https://arxiv.org/abs/1910.13970].

    - You may need to add your equations for any derived parameters that have not been tracked by the sampler in `add_derived()` (more details below).

- **Not Yet Implemented**: Return triangle plots of requested parameters using `getdist`.

- Return $\chi^2$ at Maximum Likelihood (ML) or Maximum A Posteriori (MAP) points using `getdist` (requires that you have already ran `Cobaya` with a minimizer (`ignore_prior=True` for ML and `ignore_prior=False` for MAP)).

    - You may need to add the names of the likelihoods used in your run if not listed in the code (more details below).

- Compute the AIC and DIC of models using the `get_chi2()` function.
